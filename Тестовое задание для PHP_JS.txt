Тестовое задание для FullStack Web Developer (PHP/JS)

Реализовать сервис хранения файлов. Сервис состоит минимум из 2 страниц. Страницы со списком и страницы создания и редактирования записи. Запись - это комбинация из названия и файла.

ОБЯЗАТЕЛЬНЫЙ СТЕК:
    
1.	Бэк должен быть реализован на фреймворке Laravel.
2.	Фронт должен быть реализован с использованием Vue3 (Composition API) или React 18 (с использованием эффектов).
3.	В качестве базы данных должна использоваться MySQL или PostgreSQL.

ДОПОЛНИТЕЛЬНЫЕ ИНСТРУМЕНТЫ:

1.	Инструменты для верстки можно выбрать любые: Bootstrap, Material, Tailwind и т. д.
2.	Желательно использовать один или несколько дополнительных инструментов:
2.1.	Inertia.js.
2.2.	Vuex, Vue Router.
2.3.	Redux, Redux Toolkit, React Router.
2.4.	Axios, Lodash.
2.5.	SASS, SCSS, LESS.

НЕОБХОДИМО РЕАЛИЗОВАТЬ:

1.	Страница со списком.
1.1.	Список файлов в табличном или блочном виде.
1.2.	Пагинация с лимитом по 50 записей на одной странице.
1.3.	В списке файлов должны быть выведены:
1.3.1.	Название введённое пользователем. Если оно не указано, то оригинальное название файла.
1.3.2.	Размер файла в мегабайтах.
1.3.3.	Расширение файла.
1.3.4.	Для изображений должно выводиться кропнутое превью размером 100 x 100.
1.3.5.	Ссылка на скачивание оригинального файла.
1.3.6.	Общее количество записей и количество записей на текущей странице.
1.4.	В списке должны быть ссылки для перехода на страницу редактирования и удаления записи. Удаление записи должно быть реализовано через компонент модального окна с подтверждением и отменой. Удаляться должна не только запись в БД, но и сам файл в хранилище.
1.5.	Поиск по названиям файлов. Параметры поиска должны сохраняться в query string браузера и открываться по прямой ссылке. Пример: http://localhost?search=название файла
2.	Страница создания и редактирования.
2.1.	Не обязательное поле "Название файла" input[type=text].
2.2.	Поле для загрузки файла. Это поле должно быть обязательным только при загрузке нового или обновлении существующего файла. Будет плюсом стилизация input[type=file] и реализация этого компонента с использованием drag-and-drop.
2.3.	Поля формы название и файл реализовать в отдельных компонентах.
2.4.	Кнопки "Сохранить" и "Удалить". При нажатии на обе эти кнопки должно открываться модальное окно с подтверждением действия.
2.5.	Для функционала загрузки файла должны быть реализованы:
2.5.1.	Валидация на максимальный размер файла в 8 мегабайт.
2.5.2.	Статус бар с прогрессом загрузки файла.


P.S. Задание будет проверяться как на логику самого кода, так и на чистоту вёрстки UI.

Проект с решением надо скинуть в виде ссылки на публичный репозиторий (GitHub)
